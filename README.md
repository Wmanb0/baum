## 德语词汇关系网 — 功能与用法说明，记得保存JSON就不会丢数据

### 这是什么

一个纯前端单页应用(HTML + JS + vis-network),数据存在浏览器 localStorage,无后端、无网络请求。定位是**德语词汇的关系图谱整理工具**,不是背单词软件。核心目标:把学过的词组织成一张网络,标注词与词之间的关系(近义、派生、易混淆等),而不是孤立记忆。

技术栈:单个 `index.html` + `app.js` + 本地 `vis-network.min.js`。数据模型有 5 类实体:`words`(词)、`edges`(词间关系)、`topics`(专题)、`confusion_groups`(易混淆组)、`settings`(含图谱节点坐标)。

------

### 数据模型

**词 (word):** 德语词、原形 lemma、词性、等级(A1/A2/B1/B2)、中文/英文释义、语法信息(冠词、复数、可分、前缀、二分词、过去式、助动词、支配格)、固定搭配、例句、备注,以及两个标记位 `is_key`(重点整理词)和 `needs_contrast`(需要区别说明)。

**关系 (edge):** 连接两个词,有 10 种类型(近义、词根相关、前缀派生、正式程度、属于专题、易混淆、固定搭配、反义、语义相关、中文近似),有方向属性:

- **无向 (undirected):** 对称关系,如近义词。图上无箭头。
- **有向 (directed):** 单向延伸关系,如 fassen → umfassen(派生),箭头从起点指向终点。

**专题 (topic):** 词的分类,支持父子层级。

**易混淆组 (confusion_group):** 一组容易混淆的词 + 每个词的对比说明(中文近似、使用场景、搭配、对比例句)。

------

### 整理状态(自动计算,非手动设置)

每个词的状态由其内容自动推导,共 6 级,从低到高:

| 状态              | 条件                         |
| ----------------- | ---------------------------- |
| 未整理 raw        | 什么都没有                   |
| 已解释 explained  | 有中文释义                   |
| 已归类 topic_only | 加入了专题                   |
| 已连接 linked     | 有关系边                     |
| 已写区别 compared | 有关系 + 有区别说明          |
| 整理完成 complete | 有例句 + 有关系 + 有区别说明 |

图谱节点颜色即代表状态。"缺失项"(缺中文、缺专题、缺连接、缺例句、需区别但未写)比状态标签更重要,是驱动整理的核心提示。

------

### 8 个页面

#### 1. 大词云(图谱主页)

vis-network 渲染的关系网络。节点=词(圆点,颜色=状态,大小=关系数)和专题(方块)。边=关系(样式区分类型,箭头区分方向)。

左上浮动工具栏(可收起):

- **搜索:** 德语/中文/专题名,命中节点白框高亮。
- **视图切换:** 全部网络 / 当前词周围 / 按专题看 / 待串联词 / 易混淆组 / 孤立词。
- **筛选:** 等级、状态、关系类型、专题。
- **显示间接关系:** 见下方"间接关系"说明。
- **重置视图:** 回到全局并 fit 全貌。

点节点 → 右侧详情面板显示该词全部信息 + 快速操作按钮。点边 → 显示关系详情(可切换方向、改说明、删除)。

**在图上直接连边:** 点某词详情里的"➕ 在图上连边",再点目标词,弹出内联表单选关系类型、方向、说明,保存。连一条后保持连边模式,可继续从同一起点连下一个。适合一个词根派生多个词的场景。

**以此词为中心查看:** 显示该词能通过关系链到达的**全部**词(不限步长)。有向边沿方向可达,无向边双向可达。例:fassen → umfassen → enthalten,点 fassen 能看到 enthalten。此操作不改变缩放。

#### 2. 添加词条

录入/编辑词。双列布局,顶部有 sticky 操作条(保存/清空/删除按钮常驻,任意滚动位置可点)。含基础信息、语法信息、例句、串联信息四组。

串联信息里可加关系:输入相关旧词时出现最邻近候选(自定义 autocomplete,只显示前 6 条),点选后设关系类型、方向、说明保存。

#### 3. 待串联词

所有尚未"整理完成"的词(全等级),按缺失项驱动整理。带搜索、等级筛选、分页。每个词列出缺什么 + 推荐下一步操作按钮。

#### 4. 专题

创建/删除专题,支持父子层级。展示每个专题下的词(颜色标状态,点击跳编辑)。

#### 5. 易混淆组

创建组、加词、逐词填写对比说明(中文近似/使用场景/搭配/对比例句),以对比表呈现。带搜索、分页。可跳到图谱查看本组。

#### 6. 串联检查(质量控制中心)

诊断词网质量,6 类问题:孤立词、只有专题无连接、有关系但无说明、需区别但未写、未完成整理、可能可连接的词(启发式建议)。每类带搜索和分页。"可连接建议"按需点击计算(词量大时较重)。

#### 7. 词库

全部词的表格,可搜索、按列排序、分页。列含单词/词性/等级/中文/状态/专题/关系数/需对比/缺失项。

#### 8. 数据

导出/导入 JSON 备份。导入时做 schema 校验、字段补全、非法值清洗、旧版本 ID 迁移,并报告修正/丢弃项。含清空为空库。

------

### 间接关系(图谱特性)

当你**过滤视图**(如切到 B1)时,某些中间词被隐藏,导致两个可见词之间断连。开启"显示间接关系"后,系统会补上间接连线:仅当两个可见词之间存在一条**全部由隐藏词构成的路径**时才画。

例:greifen → fassen → umfassen,若 fassen 非 B1 被隐藏,B1 视图下 greifen 和 umfassen 会用间接连线(点虚线、暗黄色)连起来。方向传递:整条链同向则间接边有向,否则无向。点间接边可查看被隐藏的中间路径。

间接边是渲染期推导产物,不写入数据、不导出,切回全局或关开关即消失。可见节点过多(>600)时自动跳过以防卡顿。

------

### 主题与其他

- **深色/浅色主题:** 右上角切换,存 localStorage。
- **未导出提示:** 标题栏显示改动是否已导出到文件(localStorage 是自动的,但文件备份需手动导出)。
- **数据安全:** 所有数据在本地浏览器,清缓存会丢失。定期用"数据"页导出 JSON 备份。

------

### 典型工作流

1. 添加词条录入新词(至少填德语词 + 中文)。
2. 保存后立即在串联信息里连到已有的相关词,选好方向。
3. 需要对比的词在易混淆组里写清区别。
4. 定期看**待串联词**清空缺失项,看**串联检查**发现孤立词和可连接建议。
5. 在**大词云**用"以此词为中心"审视某个词根/主题的关系子网。
6. 定期在**数据**页导出 JSON 备份。
